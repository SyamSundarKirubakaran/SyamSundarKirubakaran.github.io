Since I'll have to show the update check on the action bar if the contents in the text view changes Observables has to be used. Currently before my PR there were three Observables entries for three fields namely for the Name, username and Email
It's a matter of fact that Obervables.combineLatest can only take in three inline parameter but I'll have to listen for changes in 4 fields, the 4th field being Avatar URL and hence I splitted the Obervables.combineLatest into two each listening for changes in two fields individually.

@filipe.brito suggested to find a better solution for this so that the code gets even more consistent and Hence he commented on making some changes in my PR.


As suggested by @filipe.brito, I made the changes to listen to changes for 4 fields at a time using Observables and made commits on the current PR.
